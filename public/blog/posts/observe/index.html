Vue Data Hijacking Explained -- The Observer Pattern

27 Aug, 2019
|

From the diagram above, we can see that Vue uses Object.defineProperty() for data hijacking. It also wraps a relay station layer for data change propagation, namely the Dep and Watcher classes we see.

Data Hijacking

Recursive Traversal

Vue hijacks all the data inside data, which requires traversing the object to complete the hijacking of each property. The relevant source code is as follows:

walk (obj: Object) {
  const keys = Object.keys(obj)
  // 遍历将其变成 vue 的访问器属性
  for (let i = 0; i < keys.length; i++) {
    defineReactive(obj, keys[i], obj[keys[i]])
  }
}

Publish/Subscribe

From the object traversal above, we saw defineReactive. The key point of hijacking lies in this function, which encapsulates getter and setter functions using the observer pattern for mutual monitoring.

// 设置为访问器属性，并在其 getter 和 setter 函数中，使用发布/订阅模式，互相监听。
export function defineReactive (
  obj: Object,
  key: string,
  val: any
) {
  // 这里用到了观察者（发布/订阅）模式进行了劫持封装，它定义了一种一对多的关系，让多个观察者监听一个主题对象，这个主题对象的状态发生改变时会通知所有观察者对象，观察者对象就可以更新自己的状态。
  // 实例化一个主题对象，对象中有空的观察者列表
  const dep = new Dep()

  // 获取属性描述符对象(更多的为了 computed 里面的自定义 get 和 set 进行的设计)
  const property = Object.getOwnPropertyDescriptor(obj, key)
  if (property && property.configurable === false) {
    return
  }

  const getter = property && property.get
  const setter = property && property.set

  let childOb = observe(val)
  Object.defineProperty(obj, key, {
    enumerable: true,
    configurable: true,
    // 收集依赖，建立一对多的的关系，让多个观察者监听当前主题对象
    get: function reactiveGetter () {
      const value = getter ? getter.call(obj) : val
      if (Dep.target) {
        dep.depend()
        if (childOb) {
          childOb.dep.depend()
          // 这里是对数组进行劫持
          if (Array.isArray(value)) {
            dependArray(value)
          }
        }
      }
      return value
    },
    // 劫持到数据变更，并发布消息进行通知
    set: function reactiveSetter (newVal) {
      const value = getter ? getter.call(obj) : val
      if (newVal === value || (newVal !== newVal && value !== value)) {
        return
      }
      if (setter) {
        setter.call(obj, newVal)
      } else {
        val = newVal
      }
      childOb = observe(newVal)
      dep.notify()
    }
  })
}

Returning an Observer Instance

return new Observer(value)

Message Encapsulation — Implementing the “Relay Station”

After collecting subscribers, spy A only needs to publish messages, while agent B and other agents only need to subscribe to messages and perform their corresponding update operations. Each module ensures its independence, achieving the two principles of high cohesion and low coupling.

Let’s jump straight into how Vue handles message encapsulation.

Dep

Dep, short for Dependency. From the name, we can roughly tell that the Dep class is used for dependency collection. But how exactly does it collect? Let’s look at the source code directly.

let uid = 0

export default class Dep {
  static target: ?Watcher;
  id: number;
  subs: Array<Watcher>;

  constructor () {
    // 用来给每个订阅者 Watcher 做唯一标识符，防止重复收集
    this.id = uid++
    // 定义subs数组，用来做依赖收集(收集所有的订阅者 Watcher)
    this.subs = []
  }

  // 收集订阅者
  addSub (sub: Watcher) {
    this.subs.push(sub)
  }

  depend () {
    if (Dep.target) {
      Dep.target.addDep(this)
    }
  }

  notify () {
    // stabilize the subscriber list first
    const subs = this.subs.slice()
    for (let i = 0, l = subs.length; i < l; i++) {
      subs[i].update()
    }
  }
}

// the current target watcher being evaluated.
// this is globally unique because there could be only one
// watcher being evaluated at any time.
Dep.target = null
  1. Define a subs array to collect subscriber Watchers
  2. When a data change is hijacked, notify subscriber Watchers to perform update operations

The source code also exports two methods to manipulate Dep.target, as follows:

// 定义收集目标栈
const targetStack = []

export function pushTarget (_target: Watcher) {
  if (Dep.target) targetStack.push(Dep.target)
  // 改变目标指向
  Dep.target = _target
}

export function popTarget () {
  // 删除当前目标，重算指向
  Dep.target = targetStack.pop()
}

Watcher

Watcher means observer. Its responsibility is to subscribe to Dep. When Dep sends a message (notify), all Watchers subscribed to that Dep will perform their own update operation.

export default class Watcher {
  vm: Component;
  expression: string;
  cb: Function;

  constructor (
    vm: Component,
    expOrFn: string | Function,
    cb: Function,
    options?: Object
  ) {
    this.vm = vm
    vm._watchers.push(this)
    this.cb = cb
    // parse expression for getter
    if (typeof expOrFn === 'function') {
      this.getter = expOrFn
    } else {
      // 解析表达式
      this.getter = parsePath(expOrFn)
      if (!this.getter) {
        this.getter = function () {}
      }
    }
    this.value = this.get()
  }

  get () {
    // 将目标收集到目标栈
    pushTarget(this)
    const vm = this.vm

    let value = this.getter.call(vm, vm)
    // 删除目标
    popTarget()

    return value
  }

  // 订阅 Dep，同时让 Dep 知道自己订阅着它
  addDep (dep: Dep) {
    const id = dep.id
    if (!this.newDepIds.has(id)) {
      this.newDepIds.add(id)
      this.newDeps.push(dep)
      if (!this.depIds.has(id)) {
        // 收集订阅者
        dep.addSub(this)
      }
    }
  }

  // 订阅者'消费'动作，当接收到变更时则会执行
  update () {
    this.run()
  }

  run () {
    const value = this.get()
    const oldValue = this.value
    this.value = value
    this.cb.call(this.vm, value, oldValue)
  }
}

Now when we look at Dep and Watcher again, we need to understand two key points:

  1. Dep is responsible for collecting all subscriber Watchers. It doesn’t care about who specifically or how many there are — it only needs to collect Watchers that subscribe to its messages through the target pointer, and then handle message publishing via notify.
  2. Watcher is responsible for subscribing to Dep, letting Dep collect it during subscription, and performing its update operation when receiving a message published by Dep.

The two may seem mutually dependent, but they actually maintain their independence, ensuring the single responsibility of each module.

Vue Event Propagation Also Follows the Observer Pattern

Summary

  • The Observe class checks whether the value is an array. Arrays use observeArray to hijack first-level properties, while objects call walk to iterate over keys and call defineReactive for hijacking.
  • defineReactive creates a Dep instance and uses Object.defineProperty(obj, key, {props}) to modify get and set.
  • Dep has its own id and an array subs for dependency collection of Watcher instances.
  • Watcher uses the addDep method to add a Dep, and calls addSub to add the Watcher to the subs array.
  • On obj.set, Dep.notify() is called, which iterates over the Dep subs array and calls sub[i] (watcher).update.

Frontend Knowledge System - Runtime Environment

6 Sep, 2019
|

We need to clarify the relationship between language and environment:

ECMAScript describes the syntax and basic object specifications of the JavaScript language. The browser, as one runtime environment for JavaScript, provides: the Document Object Model (DOM), which describes methods and interfaces for handling web page content; and the Browser Object Model (BOM), which describes methods and interfaces for interacting with the browser. Node is also a runtime environment for JavaScript, providing APIs for I/O operations, networking, and more.

Browser APIs

  1. Browser-provided DOM manipulation APIs conforming to W3C standards, browser differences, and compatibility
  2. All global APIs provided by the Browser Object Model (BOM), browser differences, and compatibility
  3. Performance optimization for heavy DOM manipulation and large datasets (batch operations, Diff, requestAnimationFrame, etc.) *
  4. Browser large-scale data storage and operation performance optimization localStorage sessionStorage distributed cluster deployment
  5. Specific implementation of DOM event flow, differences across browsers, event delegation capture, target, bubbling; target returns the node that triggered the event; currentTarget returns the element whose listener triggered the event; IE attachEvent; addEventListener third parameter changed to options: capture, once (removed after call), passive (will never call preventDefault())
  6. Various ways the frontend initiates network requests and their underlying implementations; ability to hand-write native ajax, fetch; proficiency with third-party libraries * fetch('url',{method: 'post',body:{},headers:{'Content-type':"application/json;charset=utf-8"}}) xhr, ajax
  7. Browser same-origin policy, how to bypass it, differences between approaches and how to choose * Same domain, same protocol, same port; JSONP (script tag, GET only); postMessage; WebSocket; document.domain for subdomains
  8. Browser storage mechanisms, their pros and cons, and making the right choice in development * cookie 4KB, sent with requests; localStorage permanent storage; sessionStorage cleared when tab closes; IndexedDB
  9. Cross-tab communication in the browser * WebSocket, localStorage, SharedWorker (same-origin)

Browser Internals *

  1. JavaScript engines used by different browsers, their differences, and how to detect them in code navigator.userAgent
  2. How many interactions occur with the server from request start to request end * 3+2+4
  3. Detailed description of the process from entering a URL to page rendering ** From URL to Page Rendering
  4. How the browser parses HTML code and the DOM tree construction process * HTML parse -> build DOM; CSS parse -> build CSSOM; JS blocks DOM; tokenization and tree construction
  5. How the browser parses CSS rules and applies them to the DOM tree * Right to left; different selector specificity weights; inline > id > class/pseudo-class/attribute > element/pseudo-element
  6. How the browser paints the styled DOM tree * Build render tree -> reflow -> repaint
  7. Browser execution mechanism, how to configure synchronous and asynchronous resource loading * CSS loading blocks rendering but not DOM construction; CSS parsing blocks JS execution (must wait for CSSOM); HTML is parsed and displayed incrementally; CSS must be fully parsed before entering render tree phase; JS fully blocks; split into smaller chunks; use async, preload (tells the browser the resource is definitely needed, browser will load it), prefetch (tells the browser the resource might be needed, browser may not load it)
  8. Underlying principles of browser reflow and repaint, causes, and how to effectively avoid them *
  9. Browser garbage collection mechanism, how to avoid memory leaks ** Set unused variables to null; avoid defining unnecessary global variables; avoid circular dependencies
  10. Caching strategies used by browsers, how to choose and control the appropriate caching strategy ** Browser Caching Mechanism

Node

Node source code reading

  1. Understanding Node’s role in applications; ability to use Node to set up frontend development environments, manipulate files, operate databases, etc.
  2. Mastery of at least one Node framework such as Express; understanding the differences between Express and Koa
  3. Proficiency with Node’s built-in APIs such as Path, Http, Child Process, and understanding their underlying implementations
  4. Node’s event loop and its similarities and differences with the browser’s event loop

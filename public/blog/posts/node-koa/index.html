Node -- Koa

23 May, 2019
|

After comparing several popular Node.js frameworks, I found that Koa’s design philosophy is the one I appreciate the most:

  • The codebase is only about 1,000 lines
  • Rich plugin ecosystem

This article follows along with Ruan Yifeng’s tutorial to set up a simple Koa environment.

Basic Usage

const Koa = require('koa');
const app = new Koa();

const main = ctx => {
  ctx.response.body = 'Hello World';
};

app.use(main);
app.listen(3000);

Alright, that’s the end of this article. Haha, just kidding.

Handling Requests

Most of the time, I use Node.js to handle requests.

const Koa = require('koa');
const app = new Koa();
const main = ctx => {
  if (ctx.request.url.indexOf('test.html') !== -1) {
    ctx.response.type = 'html';
    ctx.response.body = '<a href="/">Index Page</a>';
  } else {
    ctx.response.body = 'Hello World';
  }
};

app.use(main);
app.listen(3000);

This makes it very convenient to handle requests and design responses.

Middleware

Koa’s most distinctive and important design feature is middleware. To understand middleware, let’s first look at how to implement a Logger (logging) feature.

const logger = (ctx, next) => {
  console.log(`${Date.now()} ${ctx.request.method} ${ctx.request.url}`);
  next();
}
app.use(logger);

A function like the logger above is called “middleware” because it sits between the HTTP Request and HTTP Response, implementing some intermediate functionality. app.use() is used to load middleware.

Essentially, all of Koa’s functionality is implemented through middleware. The main function in the earlier examples is also middleware. Each middleware accepts two parameters by default: the first is the Context object, and the second is the next function. Simply calling the next function passes execution control to the next middleware.

Middleware Stack

Multiple middleware form a stack structure (middleware stack), executed in “first-in-last-out” order.

  • The outermost middleware executes first.
  • Calling the next function passes execution to the next middleware.
  • …
  • The innermost middleware executes last.
  • After execution completes, control is passed back to the previous layer’s middleware.
  • …
  • After the outermost middleware regains control, it executes the code after the next function call.
const one = (ctx, next) => {
  console.log('>> one');
  next();
  console.log('<< one');
}

const two = (ctx, next) => {
  console.log('>> two');
  next();
  console.log('<< two');
}

const three = (ctx, next) => {
  console.log('>> three');
  next();
  console.log('<< three');
}

app.use(one);
app.use(two);
app.use(three);
//result
>> one
>> two
>> three
<< three
<< two
<< one

Error Handling in Requests

Basic Approach

To handle errors conveniently, it’s best to catch them using try…catch. However, writing try…catch for every middleware is too tedious. Instead, we can have the outermost middleware handle errors for all middleware.

const handler = async (ctx, next) => {
  try {
    await next();
  } catch (err) {
    ctx.response.status = err.statusCode || err.status || 500;
    ctx.response.body = {
      message: err.message
    };
  }
};

const main = ctx => {
  ctx.throw(500);
};

app.use(handler);
app.use(main);

Listening for the Error Event

If an error occurs during runtime, Koa will emit an error event. Listening for this event is another way to handle errors.

const main = ctx => {
  ctx.throw(500);
};

app.on('error', (err, ctx) =>
  console.error('server error', err);
);

Emitting the Error Event

It’s important to note that if an error is caught by try…catch, the error event will not be triggered. In this case, you must call ctx.app.emit() to manually emit the error event for the listener to work.

const handler = async (ctx, next) => {
  try {
    await next();
  } catch (err) {
    ctx.response.status = err.statusCode || err.status || 500;
    ctx.response.type = 'html';
    ctx.response.body = '<p>Something wrong, please contact administrator.</p>';
    ctx.app.emit('error', err, ctx);
  }
};

const main = ctx => {
  ctx.throw(500);
};

app.on('error', function(err) {
  console.log('logging error ', err.message);
  console.log(err);
});

In the code above, the main function throws an error that is caught by the handler function. Inside the catch block, ctx.app.emit() is used to manually emit the error event so that the listener can detect it.

Useful Plugins

@koa/cors

This plugin handles cross-origin requests: @koa/cors

/**
 * CORS middleware
 *
 * @param {Object} [options]
 *  - {String|Function(ctx)} origin `Access-Control-Allow-Origin`, default is request Origin header
 *  - {String|Array} allowMethods `Access-Control-Allow-Methods`, default is 'GET,HEAD,PUT,POST,DELETE,PATCH'
 *  - {String|Array} exposeHeaders `Access-Control-Expose-Headers`
 *  - {String|Array} allowHeaders `Access-Control-Allow-Headers`
 *  - {String|Number} maxAge `Access-Control-Max-Age` in seconds
 *  - {Boolean} credentials `Access-Control-Allow-Credentials`
 *  - {Boolean} keepHeadersOnError Add set headers to `err.header` if an error is thrown
 * @return {Function} cors middleware
 * @api public
 */

const Koa = require('koa');
const cors = require('@koa/cors');
const app = new Koa();
app.use(cors({credentials: true}));

koa-static

This plugin serves static assets: koa-static

const path = require('path');
const serve = require('koa-static');

const main = serve(path.join(__dirname));
app.use(main);

koa-compose

This plugin composes multiple middleware into one: koa-compose

const compose = require('koa-compose');

const logger = (ctx, next) => {
  console.log(`${Date.now()} ${ctx.request.method} ${ctx.request.url}`);
  next();
}

const main = ctx => {
  ctx.response.body = 'Hello World';
};

const middlewares = compose([logger, main]);
app.use(middlewares);

koa-body

This plugin converts POST request data into key-value pairs and can handle file uploads.

POST key-value pairs

const koaBody = require('koa-body');

const main = async function(ctx) {
  const body = ctx.request.body;
  if (!body.name) ctx.throw(400, '.name required');
  ctx.body = { name: body.name };
};

app.use(koaBody());

File upload

const os = require('os');
const path = require('path');
const koaBody = require('koa-body');

const main = async function(ctx) {
  const tmpdir = os.tmpdir();
  const filePaths = [];
  const files = ctx.request.body.files || {};

  for (let key in files) {
    const file = files[key];
    const filePath = path.join(tmpdir, file.name);
    const reader = fs.createReadStream(file.path);
    const writer = fs.createWriteStream(filePath);
    reader.pipe(writer);
    filePaths.push(filePath);
  }

  ctx.body = filePaths;
};

app.use(koaBody({ multipart: true }));
